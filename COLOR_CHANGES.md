# TanjaMol Color Changes

This file documents the storefront color updates made during the customer-facing polish pass.

## Core Theme Tokens

The main store accent shifted from green to warm orange:

- Primary brand: `#00a66c` -> `#ff9900`
- Strong brand/accent: `#0f7d55` -> `#b45309`
- Soft brand background: `#e9f8ef` -> `#fff3df`
- Focus color: `#00d084` -> `#ffb84d`
- Brand shadow: green shadow -> orange shadow

## Header And Dark Surfaces

The dark header color was standardized:

- Header: `#102118` -> `#131921`
- Header alpha: `rgba(16, 33, 24, 0.96)` -> `rgba(19, 25, 33, 0.96)`

This also affects dark panels, product hero sections, policy strips, menu panels, and dark CTA states that use the shared header color.

## Surface Colors

The light tinted surface became more neutral:

- Tinted surface: `#f2f7f4` -> `#f8fafc`

This reduces the green cast in product detail blocks, cards, specs, and soft UI panels.

## Product Page Highlights

The product page now uses the warm brand accent consistently:

- Active gallery dots: green -> orange
- Active desktop gallery border: green -> orange
- Product badge background: green-tinted -> warm brand-tinted
- Product price: green -> warm orange/brown accent
- Selected variant border/background/text: green style -> warm brand style
- “Important” labels and related section accent: green -> warm brand accent

## Motion And Feedback Colors

Add-to-cart feedback now uses a warmer visual language:

- Add spark main dot: green -> amber
- Additional cart-feedback dot remains green as a success signal
- Mobile hero dark gradient now matches the standardized header color

## Admin Color Changes To Review Later

Some shared color token changes naturally affect admin/login surfaces that use global storefront tokens or shared components. The admin/dashboard polish phase should review these deliberately before finalizing admin colors.
